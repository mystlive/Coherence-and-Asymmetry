## 1. 概要と全体像

この論文は **2 チャンネルの脳波 (EEG1, EEG2) から、左右の脳の関係性を周波数別に数値化する** ための 2 つの指標を定義しています。

| 指標 | 何を測るか | 状況 |
|---|---|---|
| **Coherence** (γ²) | 2 チャンネルの**位相同期の強さ** | 「左右の脳が同じリズムで動いているか」 |
| **Asymmetry** (A) | 2 チャンネルの**パワー（強度）の偏り** | 「どちらの脳が強く活動しているか」 |

両指標は **独立した情報** を持ちます。例えば：

- **高 Coherence × 低 Asymmetry**：左右が同じリズムで同じ強さに動いている（理想的な協調）
- **高 Coherence × 高 Asymmetry**：同じリズムだが片側が強い（同期はしているが非対称な活動）
- **低 Coherence × 低 Asymmetry**：左右の活動が独立していて、たまたま強度は同じ
- **低 Coherence × 高 Asymmetry**：完全にバラバラに動いている

このため、サイマトロン系装置では **両方を表示する** のが標準です。

## 2. 基本概念の整理

### 2.1 信号と FFT

時間領域の信号を周波数領域に変換するのが FFT (Fast Fourier Transform)。

$$x(t) \;\xrightarrow{FFT}\; X(f) = \operatorname{Re}_x(f) + j\,\operatorname{Im}_x(f)$$

- \operatorname{Re}_x(f) : 実部 (cos 成分)
- \operatorname{Im}_x(f) : 虚部 (sin 成分)
- j : 虚数単位 (j^2 = -1)

### 2.2 複素数の絶対値

$$|\Re + j\Im|^2 = \Re^2 + \Im^2$$

これが論文の冒頭にある「絶対値の二乗 = 実部² + 虚部²」の意味です。

### 2.3 オートパワースペクトル (Sxx)

ある信号が「周波数 f にどれだけのエネルギーを持つか」を表します。

$$S_{xx}(f) = X^*(f) · X(f) = \operatorname{Re}_x^2(f) + \operatorname{Im}_x^2(f)$$

ここで X* は複素共役 (実部はそのまま、虚部の符号を反転)。

> **メモ:** 論文中の `Power = Sxx / 2` は片側スペクトル換算の慣例です。両側スペクトルから片側に圧縮する際の係数です。

### 2.4 クロスパワースペクトル (Sxy)

2 つの信号の **「周波数 f における関係性」** を複素数で表現したもの。

$$S_{xy}(f) = X^*(f) · Y(f)$$

実部・虚部に展開すると：

$$S_{xy}(f) = (\operatorname{Re}_x + j\operatorname{Im}_x)^* · (\operatorname{Re}_y + j\operatorname{Im}_y)$$

$$= (\operatorname{Re}_x - j\operatorname{Im}_x) · (\operatorname{Re}_y + j\operatorname{Im}_y)$$

$$= (\operatorname{Re}_x\operatorname{Re}_y + \operatorname{Im}_x\operatorname{Im}_y) + j(\operatorname{Re}_x\operatorname{Im}_y - \operatorname{Im}_x\operatorname{Re}_y)$$

> **重要:** 論文には `Sxy = (Re_x + jIm_x) * (Re_y - jIm_y)` とありますが、これは X · Y* の形。X* · Y と X · Y* は互いに複素共役の関係で、後の Coherence 計算では二乗するため **どちらでも同じ結果**になります。

クロスパワースペクトルの **絶対値の二乗** は：

$$|S_{xy}(f)|^2 = \operatorname{Re}_{xy}^2 + \operatorname{Im}_{xy}^2$$

これが Coherence の分子です。

## 3. Coherence（コヒーレンス）の数式

### 3.1 公式

論文の定義：

$$\boxed{\;\gamma^2(f) = \frac{|S_{xy}(f)|^2}{S_{xx}(f) · S_{yy}(f)}\;}$$

- 値の範囲: 0 ≤ γ² ≤ 1
- 0 = 無相関、1 = 完全な線形連動（位相も振幅も揃って動く）

### 3.2 何を表しているか

**「2 つの信号がその周波数でどれだけ一緒に動いているか」** を 0〜1 で表します。

- γ² = 1.0: その周波数では片方の信号からもう片方を完全に予測できる
- γ² = 0.5: 半分くらい連動している
- γ² = 0.0: その周波数では完全に独立に動いている

### 3.3 数学的に重要な性質

**1 セグメントだけで計算すると、γ² は周波数によらず常に 1.0 になります。**

これは数学的な恒等式であり、バグではなく定義上そうなります：

$$|S_{xy}|^2 = |X^* Y|^2 = |X|^2 · |Y|^2 = S_{xx} · S_{yy}$$

$$\therefore \gamma^2 = \frac{S_{xx} · S_{yy}}{S_{xx} · S_{yy}} = 1$$

**この性質こそが、論文が「複数セグメントで平均しろ」と強調する理由です。**

意味のある Coherence を得るには、複数の独立した観測区間（セグメント）にわたって **クロススペクトルの実部・虚部を平均** し、その平均から計算する必要があります（[セクション 6](#6-致命的な罠なぜ平均化の順序が全てを決めるか) 参照）。

## 4. Asymmetry（非対称性）の数式

### 4.1 公式

論文の定義：

$$\boxed{\;A(f) = 2 · \frac{|S_{xx}(f) - S_{yy}(f)|}{S_{xx}(f) + S_{yy}(f)}\;}$$

- 値の範囲: 0 ≤ A ≤ 2 (論文では 0% 〜 200% と表記)
- 0 = 左右パワーが完全に対称
- 2 = 完全に片側のみにエネルギーがある

### 4.2 何を表しているか

**「2 つの信号のうち、どちらにどれだけエネルギーが偏っているか」** を表します。

| Sxx : Syy の比 | A の値 | % 表記 |
|---|---|---|
| 1 : 1（対称） | 0.000 | 0% |
| 2 : 1 | 0.667 | 67% |
| 3 : 1 | 1.000 | 100% |
| 9 : 1 | 1.600 | 160% |
| 1 : 0（片側のみ） | 2.000 | 200% |

### 4.3 係数2が付属する理由

係数2がない場合、最大値は 1.0 になります。論文の定義では係数2を掛けることで、

- 0% = 完全対称
- 100% = ある程度の偏り（パワー比 3:1）
- 200% = 完全に片側

という、**Coherence と桁を揃えた表記** になっています。Coherence が 0〜1 (0%〜100%)、Asymmetry が 0〜2 (0%〜200%) で、両者を %表記で並べると同じスケール感で比較できる工夫が入っていました。

### 4.4 Coherence との違い（再掲）

| | Coherence | Asymmetry |
|---|---|---|
| 入力 | クロススペクトル Sxy | オートスペクトルのみ Sxx, Syy |
| 位相情報 | **使う**（複素数） | **使わない**（実数のパワーのみ） |
| 測るもの | 連動の強さ | 強度の偏り |

## 5. 計算手順 — 論文 5 ステップの精読

論文に書かれた手順を、現代的な言葉で再表記し、各ステップの意味を解説します。

### Step 1. 時間信号 x(t), y(t) を収集する

2 チャンネル分の時系列データを集める。EEG であれば左右半球の電極から。サンプリング周波数 Fs は機器設定（典型値 200 Hz）。

### Step 2. FFT で周波数領域へ変換

> 論文表記: "FFT them into X(t) and Y(t)"
> ※ これは原文のタイポで、正しくは X(f), Y(f) です。FFT の出力は周波数の関数。

各セグメント（典型的には 256 または 512 サンプル）に **窓関数** を掛けてから FFT します。窓関数は両端を滑らかに 0 にして「FFT が周期信号を仮定することによる漏れ（spectral leakage）」を防ぎます。

代表的な窓: Hann 窓 = 0.5 · (1 - cos(2πi / N))
論文の元コードの窓 = 1 - cos(2πi / N) (Hann × 2、正規化係数 1.5 で除算)

### Step 3. 重要：クロススペクトルは「実部・虚部のまま」平均する

> 原文: *"For coherence, you must use the real and imaginary parts of the cross power spectrum averaged over several segments, not the |Sxy(f)| averaged over those segments. Only after getting the average parts do you do the ℜ² + ℑ² calculation."*

これが論文全体で最も重要な指示です。

**正しい順序**:

$$\overline{S_{xy}}(f) = \frac{1}{M}\sum_{m=1}^{M} S_{xy}^{(m)}(f)$$

$$|\overline{S_{xy}}(f)|^2 = (\text{Re}\overline{S_{xy}})^2 + (\text{Im}\overline{S_{xy}})^2$$

(まず複素数を平均し、最後に絶対値の二乗を取る)

**誤った順序** (絶対NG):

$$\frac{1}{M}\sum_{m=1}^{M} |S_{xy}^{(m)}(f)|^2$$

(各セグメントで絶対値を取ってから平均)

### Step 4. オートスペクトルも同じセグメント数で平均する

$$\overline{S_{xx}}(f) = \frac{1}{M}\sum_{m=1}^{M} S_{xx}^{(m)}(f)$$

$$\overline{S_{yy}}(f) = \frac{1}{M}\sum_{m=1}^{M} S_{yy}^{(m)}(f)$$

### Step 5. 帯域集計は最後に行う

> 原文: *"Note that if you want the coherence for bands, rather than a single bin, you must do the summation at the very end, because the sum of squares (the correct way) is not the same as the square of sums."*

複数の周波数 bin を 1 つの帯域 (例: Alpha 8–13 Hz) として集計する場合：

**正しい**: 各 bin で平均クロス・オートを計算し、bin ごとに二乗和を取り、最後に帯域内で集計（または平均）。

**誤り**: 平均クロスを bin にわたって先に足し合わせ、その後二乗。これは数学的に異なる結果になります（"sum of squares" vs "square of sums"）。

## 6. 致命的な罠：なぜ「平均化の順序」が全てを決めるか

### 6.1 1 セグメントの Coherence は常に 1

[セクション 3.3](#33-数学的に重要な性質) の通り、1 セグメントだけで Coherence を計算すると **必ず 1.0** になります。

```
1 セグメントの S_xy = X*·Y
|S_xy|^2 = |X|^2 · |Y|^2 = S_xx · S_yy
γ² = S_xx · S_yy / (S_xx · S_yy) = 1
```

### 6.2 「平均後の Coherence」と「Coherence の平均」は別物

実際の数値で比較してみます（同じデータで）：

| 計算順序 | mean | max |
|---|---|---|
| **誤**: 各セグメントで γ² を計算 → 平均 | 1.000 | 1.000 |
| **正**: 実虚部のままクロスを累積 → 最後に γ² 計算 | 0.68 | 0.99 |

順序を間違えると **常に 1.0** という意味のない結果になります。

### 6.3 なぜ平均化で意味のある値になるか

複素数の平均はベクトル平均と同じです。

- **2 信号が真に同期していれば**：各セグメントの Sxy は同じ向きを持ち、平均しても向きが残り、絶対値は大きくなる。
- **2 信号が独立なら**：Sxy の向きはセグメントごとにランダム。平均するとベクトルが打ち消し合い、絶対値はほぼ 0 になる。

オートスペクトル Sxx, Syy は常に正の実数なので、向きの問題がなく、独立信号でも同期信号でも値は残ります。

結果：

- 同期信号 → |平均 Sxy|^2 ≈ 平均 Sxx · 平均 Syy → γ² ≈ 1
- 独立信号 → |平均 Sxy|^2 ≈ 0 → γ² ≈ 0

これが **位相情報を活かしてセグメント平均する Welch 法** の本質と考えます。

### 6.4 Welch 法のセグメント設計

| パラメータ | 典型値 | 影響 |
|---|---|---|
| FFT 長 n | 256, 512, 1024 | 周波数分解能 = Fs / n |
| ホップ長 (hop) | n/2 (50% overlap) | セグメント独立性とノイズ低減のバランス |
| セグメント数 M | 10〜100 | 多いほど統計的に安定（ただしデータ長が必要） |

オーバーラップを使うと有効セグメント数が増え、ノイズが下がりますが、計算量も増加します。50% オーバーラップが伝統的に最適とされます。

## 7. 帯域別解析の正しいやり方

EEG では 1 Hz 単位の議論より、生理的に意味のある **周波数帯域** で集計するのが普通です。

### 7.1 標準的な帯域定義

| 帯域 | 範囲 | 生理的意味 |
|---|---|---|
| Delta | 0.5 – 4 Hz | 深睡眠、無意識 |
| Theta | 4 – 8 Hz | 浅睡眠、瞑想、創造性 |
| Alpha | 8 – 13 Hz | 安静覚醒、リラックス |
| Beta | 13 – 30 Hz | 活動覚醒、集中、思考 |
| Gamma | 30 Hz 〜 | 高次認知、注意 |

サイマトロン Genie の帯域定義（Image 2 から確定）：

| Band | Range (Hz) |
|---|---|
| Total | 0.7 – 25.0 |
| Delta | 0.7 – 3.5 |
| Theta | 3.5 – 8.0 |
| Alpha | 8.0 – 13.0 |
| Beta | 13.0 – 25.0 |
| X Band | 4.0 – 7.5 |
| Y Band | 8.5 – 12.0 |

X Band, Y Band はサイマトロン独自のサブバンドで、Theta 中央域と Alpha 中央域を狙ったもの。

### 7.2 帯域内 Coherence の集計方法

論文の Step 5 に従い、bin 単位で計算してから帯域内で集計します。

```
擬似コード:
for each bin k in band [Lo, Hi]:
    coh[k] = (avgRe[k]² + avgIm[k]²) / (avgSxx[k] · avgSyy[k])

band_max  = max(coh[k] for k in band)
band_mean = mean(coh[k] for k in band)
```

純正アプリの `Coh%` 列は通常 **帯域内最大値** または **帯域内平均値** のいずれか。今回の実装では両方出力しています。

### 7.3 Mean Frequency

帯域内の「重心周波数」も実装可能です：

$$f_{\text{mean}} = \frac{\sum_k f_k · \overline{S_{xx}}(f_k)}{\sum_k \overline{S_{xx}}(f_k)}$$

純正アプリの `Mean Frequencies Hz:` の値はこれです。

## 8. 実装時に踏みやすい落とし穴 6 連発

実際にこの論文を実装する過程で遭遇した、典型的なバグの実例集。

### 罠 ① セグメント毎に Coherence を計算してしまう

**症状**: 結果が常に 1.000 (100%) になる。

**原因**: [セクション 6.1](#61-1-セグメントの-coherence-は常に-1) のとおり数学的恒等式。

**修正**: 累積器パターンを使う。

```csharp
// NG: 誤
foreach (segment) {
    coh = ComputeCoherence(segment);
    cohSum += coh;
}
result = cohSum / segCount;

// OK: 正
foreach (segment) {
    Accumulate(Sxx, Syy, Sxy_re, Sxy_im);  // 生スペクトルを足す
}
result = ComputeCoherenceFromAccumulated();  // 最後に1回だけ計算
```

### 罠 ② FFT twiddle テーブルの符号バグ

**症状**: 純粋正弦波を FFT に入れても、本来立つべき bin にエネルギーが集まらず散らばる。

**原因**: `cos` テーブルを半周期分しか持たず、剰余演算で第二半周期の符号反転を取り損なう。

```csharp
// NG: 半周期テーブル + % (MAXSAMP/2)
private int[] cosTbl = new int[MAXSAMP / 2];
int wi = cosTbl[(MAXSAMP/4 + k*delta) % (MAXSAMP/2)];  // 符号情報損失

// OK: 全周期テーブル + % MAXSAMP
private int[] cosTbl = new int[MAXSAMP];
int wi = cosTbl[(MAXSAMP/4 + k*delta) % MAXSAMP];
```

または、`cos` と `sin` を別テーブルにする（推奨）。

### 罠 ③ Nyquist bin の取りこぼし

**症状**: 最高周波数 (Fs/2) のパワーが常に 0。

**原因**: ループが `i < n/2` で終わり、`i = n/2` の bin を計算していない。

```csharp
// NG:
for (int i = 0; i < n/2; i++) { ... }

// OK
for (int i = 0; i <= n/2; i++) { ... }  // <= に変更
```

### 罠 ④ 整数 FFT のオーバーフロー

**症状**: FFT 後の値が異常に小さく、Coherence 計算で結果が ~0 になる。

**原因**: `int * int` で 32bit 上限を超え、ラップアラウンドで値が破壊される。

```csharp
// NG: int * int でオーバーフロー
int xr = (real[l] * wr - imag[l] * wi) / ZOOM;

// OK: long 経由で計算
long mr = (long)real[l] * wr - (long)imag[l] * wi;
int xr = (int)(mr / ZOOM);
```

または、最初から `double` にしてしまうのが最も安全。

### 罠 ⑤ サンプリング周波数の取り違え

**症状**: 帯域定義 (8–13 Hz など) の bin インデックスがずれて、別の周波数を集計してしまう。

**原因**: Fs ハードコードや機器設定との不一致。

**チェック方法**: 純正アプリのスペクトル表示で bin 幅を確認。bin 幅 × n = Fs が成立するはず。

```
例: bin 幅 0.78 Hz, n=256 なら Fs = 0.78 × 256 ≈ 200 Hz
```

### 罠 ⑥ 過剰なスムージング

**症状**: 計算ロジックは正しいのに、ピーク値が予想より低い。

**原因**: 3点移動平均などのポストプロセスが、シャープな共鳴ピークを潰している。

```csharp
// NG: FFT 長を増やすほど、平滑が peak を傷つける
cohSmooth[k] = (coh[k-1] + coh[k] + coh[k+1]) / 3.0;

// OK: FFT 長が十分なら平滑は不要
cohSmooth = coh;  // そのまま
```

整数演算で量子化ノイズが多かった時代の名残。double + 適切な FFT 長なら不要。

## 9. Genie 純正アプリの Bands 表との対応

サイマトロン Genie の `Bands EEG 1&2 (accumulated)` 画面は、本論文の指標を以下の列で表示しています。

```
Frequency Hz   Absolute μV²    Relative %    Asym %    Coh %
              EEG1   EEG2     EEG1   EEG2
Total  0.7-25.0  ...    ...    ...    ...    ...      ...
Delta  0.7- 3.5  ...    ...    ...    ...    ...      ...
Theta  3.5- 8.0  ...    ...    ...    ...    ...      ...
Alpha  8.0-13.0  ...    ...    ...    ...    ...      ...
Beta  13.0-25.0  ...    ...    ...    ...    ...      ...
X Band 4.0- 7.5  ...    ...    ...    ...    ...      ...
Y Band 8.5-12.0  ...    ...    ...    ...    ...      ...
Mean Frequencies Hz:    ...    ...
Segments: Collected-NN   Computed-NN
```

各列の計算式：

| 列 | 計算式 |
|---|---|
| **Absolute μV²** | 帯域内総和 平均 Sxx(f_k) (and Syy) |
| **Relative %** | (帯域パワー / Total 帯域パワー) × 100 |
| **Asym %** | A(f) の帯域内平均または最大 × 100 |
| **Coh %** | γ²(f) の帯域内平均または最大 × 100 |
| **Mean Frequencies Hz** | Σ f_k · 平均 Sxx(f_k) / Σ 平均 Sxx(f_k) |
| **Collected** | FFT 投入セグメント数 |
| **Computed** | アーチファクト除外後に有効に使われたセグメント数 |

実装時は `Computed = Collected − ExcludeRange に該当するセグメント数` となるよう、除外区間を Accumulate ループ内でスキップする処理を入れます（生データの `ExcludeRange` フィールドと連動）。

## 10. 結果の解釈ガイド

### 10.1 健常な覚醒安静の典型値

| 項目 | 期待値 |
|---|---|
| Alpha Coh % | 70–95%（高同期） |
| Alpha Asym % | 0–30%（低非対称） |
| Alpha Relative % | 30–60%（覚醒時はα優位） |
| Mean Frequency Hz | 9–11（α帯重心） |

### 10.2 注意すべきパターン

**Delta が異常に大きい (Relative > 50%)**
- 深睡眠状態 OR
- DC ドリフトやアーチファクトの混入 OR
- ハイパスフィルタ未適用

**Coh % が全帯域で低い (mean < 30%)**
- 電極接触不良
- 片側電極の信号品質低下
- 真に左右が独立した活動状態

**Asym % が特定帯域だけ突出**
- その帯域の生成源が片側半球に偏在
- 片側電極のアーチファクト

**MSC@Final の周波数がα帯外**
- α帯のコヒーレンスが他帯域より弱い → 覚醒度低下、注意散漫の可能性

### 10.3 Coherence × Asymmetry のマトリクス解釈

|  | 低 Asym (< 30%) | 中 Asym (30–70%) | 高 Asym (> 70%) |
|---|---|---|---|
| **高 Coh (> 70%)** | 理想的な左右協調 | 同期下の偏在活動 | 一側性活動の強い同期 |
| **中 Coh (30–70%)** | 部分的協調、健全範囲 | 典型的な日常状態 | 一側優位の部分同期 |
| **低 Coh (< 30%)** | 左右独立、対称 | 左右独立、軽度偏在 | 強い一側性活動 |

## 11. 付録：用語集

| 用語 | 説明 |
|---|---|
| **FFT** | Fast Fourier Transform。時間信号を周波数成分に分解するアルゴリズム。 |
| **bin** | FFT 出力の 1 つの周波数点。bin 幅 = Fs/n。 |
| **Nyquist 周波数** | Fs/2。これより高い周波数は理論的に再現不可能。 |
| **Welch 法** | 信号を複数のセグメントに分け、それぞれをFFTして平均することで安定なスペクトル推定を得る手法。 |
| **MSC** | Magnitude Squared Coherence の略。γ²(f) のこと。 |
| **窓関数** | FFT 前に信号に掛ける重み関数。両端を 0 に近づけて leakage を抑制。代表例：Hann、Hamming、Blackman。 |
| **leakage** | FFT が信号を周期と仮定することによる、本来の周波数からのエネルギー漏れ。 |
| **Auto-spectrum** | 1 信号のパワースペクトル (Sxx, Syy)。常に正の実数。 |
| **Cross-spectrum** | 2 信号間のスペクトル (Sxy)。複素数で位相情報を含む。 |
| **オーバーラップ** | Welch 法でセグメントを部分的に重ねること。50% が標準。 |
| **アーチファクト** | 脳活動以外の混入信号。瞬き、体動、筋電、商用電源など。 |
| **ハイパスフィルタ** | 低周波（DC ドリフトなど）をカットするフィルタ。EEG では 0.5 Hz 程度を境にすることが多い。 |
