## 1. 概要と全体像

この論文は **2 チャンネルの脳波 (EEG1, EEG2) から、左右の脳の関係性を周波数別に数値化する** ための 2 つの指標を定義しています。

| 指標                 | 何を測るか                  | 状況                  |
| ------------------ | ---------------------- | ------------------- |
| **Coherence** (γ²) | 2 チャンネルの**位相同期の強さ**    | 「左右の脳が同じリズムで動いているか」 |
| **Asymmetry** (A)  | 2 チャンネルの**パワー（強度）の偏り** | 「どちらの脳が強く活動しているか」   |

両指標は **独立した情報** を持ちます。例えば：

- **高 Coherence × 低 Asymmetry**：左右が同じリズムで同じ強さに動いている（理想的な協調）
- **高 Coherence × 高 Asymmetry**：同じリズムだが片側が強い（同期はしているが非対称な活動）
- **低 Coherence × 低 Asymmetry**：左右の活動が独立していて、たまたま強度は同じ
- **低 Coherence × 高 Asymmetry**：完全にバラバラに動いている

このため、サイマトロン系装置では **両方を表示する** のが標準です。

## 2. 基本概念の整理

### 2.1 信号と FFT

時間領域の信号を周波数領域に変換するのが FFT (Fast Fourier Transform)。

$$x(t) \xrightarrow{FFT} X(f) = \operatorname{Re}_x(f) + j\,\operatorname{Im}_x(f)$$

- $\operatorname{Re}_x(f)$ : 実部 (cos 成分)
- $\operatorname{Im}_x(f)$ : 虚部 (sin 成分)
- $j$ : 虚数単位 ($j^2 = -1$)

### 2.2 複素数の絶対値

$$|\Re + j\Im|^2 = \Re^2 + \Im^2$$

これが論文の冒頭にある「絶対値の二乗 = 実部² + 虚部²」の意味です。

> **論文表記についての注:** 論文原本には `|ℜ + jℑ| = ℜ² + ℑ²` と記載されていますが、これは左辺に二乗の表記が抜けたタイポと思われます（左辺が絶対値そのもの、右辺が二乗の和では式が成立しません）。正しくは `|ℜ + jℑ|² = ℜ² + ℑ²` です。以降の論文中の式（Sxx, |Sxy|² など）はすべてこの「二乗」表記で書かれているため、解釈に矛盾はありません。

### 2.3 オートパワースペクトル (Sxx)

ある信号が「周波数 f にどれだけのエネルギーを持つか」を表します。

$$S_{xx}(f) = X^*(f) \cdot X(f) = \operatorname{Re}_x^2(f) + \operatorname{Im}_x^2(f)$$

ここで $X^*$ は複素共役 (実部はそのまま、虚部の符号を反転)。

> **メモ:** 論文中の `Power = Sxx / 2` は、振幅の二乗 |X(f)|² = ℜ² + ℑ² を、物理的な平均パワー（時間領域の RMS² に対応する量）へ換算する係数です。正弦波 A·cos(2πf₀t) の平均パワーが A²/2 であることに由来します。片側スペクトル化（DC・Nyquist 以外の bin を ×2 する操作）とは別の概念であり、混同しないよう注意してください。なお、Coherence・Asymmetry の計算では分子分母で同じ係数が打ち消されるため、この /2 を適用するかどうかは結果に影響しません。

### 2.4 クロスパワースペクトル (Sxy)

2 つの信号の **「周波数 f における関係性」** を複素数で表現したもの。

$$S_{xy}(f) = X^*(f) \cdot Y(f)$$

実部・虚部に展開すると：

$$S_{xy}(f) = (\operatorname{Re}_x - j\operatorname{Im}_x) \cdot (\operatorname{Re}_y + j\operatorname{Im}_y)$$

$$= (\operatorname{Re}_x\operatorname{Re}_y + \operatorname{Im}_x\operatorname{Im}_y) + j(\operatorname{Re}_x\operatorname{Im}_y - \operatorname{Im}_x\operatorname{Re}_y)$$

> **重要：定義の流派について**  
> 論文は `Sxy = X · Y*` の形（Y の複素共役）で書かれていますが、本解説では学術文献（Bendat & Piersol 等）で主流の `Sxy = X* · Y` の流派で展開しています。両者は互いに複素共役の関係にあり：
> - `|Sxy|²` は両流派で完全に一致 → **Coherence の値は同じ**
> - クロス位相 `arg(Sxy)` の **符号が反転** → 位相遅れ／進みの解釈が逆になる
>
> 本論文の Coherence・Asymmetry 計算に関しては、どちらの定義を使っても結果は同一です。ただし、クロススペクトルから位相情報を別目的（例：時間遅延推定）に流用する場合は、どちらの流派で実装しているか明示してください。

クロスパワースペクトルの **絶対値の二乗** は：

$$|S_{xy}(f)|^2 = \operatorname{Re}_{xy}^2 + \operatorname{Im}_{xy}^2$$

これが Coherence の分子です。

## 3. Coherence（コヒーレンス）の数式

### 3.1 公式

論文の定義：

$$\boxed{\;\gamma^2(f) = \frac{|S_{xy}(f)|^2}{S_{xx}(f) \cdot S_{yy}(f)}\;}$$

- 値の範囲: 0 ≤ γ² ≤ 1
- 0 = 無相関、1 = 完全な線形連動（位相も振幅も揃って動く）

### 3.2 何を表しているか

**「2 つの信号がその周波数でどれだけ一緒に動いているか」** を 0〜1 で表します。

- γ² = 1.0: その周波数では片方の信号からもう片方を完全に予測できる
- γ² = 0.5: 半分くらい連動している
- γ² = 0.0: その周波数では完全に独立に動いている

### 3.3 数学的に重要な性質

**1 セグメントだけで計算すると、γ² は周波数によらず常に 1.0 になります。**

これは数学的な恒等式であり、バグではなく定義上そうなります：

$$|S_{xy}|^2 = |X^* Y|^2 = |X|^2 \cdot |Y|^2 = S_{xx} \cdot S_{yy}$$

$$\therefore \gamma^2 = \frac{S_{xx} \cdot S_{yy}}{S_{xx} \cdot S_{yy}} = 1$$

**この性質こそが、論文が「複数セグメントで平均しろ」と強調する理由です。**

意味のある Coherence を得るには、複数の独立した観測区間（セグメント）にわたって **クロススペクトルの実部・虚部を平均** し、その平均から計算する必要があります（[セクション 6](#6-致命的な罠なぜ平均化の順序が全てを決めるか) 参照）。

## 4. Asymmetry（非対称性）の数式

### 4.1 公式

論文の定義：

$$\boxed{\;A(f) = 2 \cdot \frac{|S_{xx}(f) - S_{yy}(f)|}{S_{xx}(f) + S_{yy}(f)}\;}$$

- 値の範囲: 0 ≤ A ≤ 2 (論文では 0% 〜 200% と表記)
- 0 = 左右パワーが完全に対称
- 2 = 完全に片側のみにエネルギーがある

### 4.2 何を表しているか

**「2 つの信号のうち、どちらにどれだけエネルギーが偏っているか」** を表します。

| Sxx : Syy の比 | A の値  | % 表記 |
| ------------ | ----- | ---- |
| 1 : 1（対称）    | 0.000 | 0%   |
| 2 : 1        | 0.667 | 67%  |
| 3 : 1        | 1.000 | 100% |
| 9 : 1        | 1.600 | 160% |
| 1 : 0（片側のみ）  | 2.000 | 200% |

### 4.3 係数2が付属する理由

係数2がない場合、`|Sxx − Syy|/(Sxx + Syy)` の値域は 0〜1（完全対称〜完全片側）となります。論文は係数2を掛けて値域を 0〜2（0%〜200%）に拡張していますが、その意図は論文中に明示されていません。考えられる理由としては：

- **直感的な閾値設定**: 係数2を掛けると「パワー比 3:1」が丁度 1.0（100%）になり、「100% を超えたら強い偏り」という読み方ができる
- **小さな非対称の視認性向上**: 0〜1 より 0〜2 の方がダイナミックレンジが広く、グラフ表示や閾値判定がしやすい

なお Coherence（0〜1）と Asymmetry（0〜2）は値域が異なるため、% 表記で並べてもスケールが揃うわけではない点に注意してください。

### 4.4 Coherence との違い（再掲）

|      | Coherence    | Asymmetry           |
| ---- | ------------ | ------------------- |
| 入力   | クロススペクトル Sxy | オートスペクトルのみ Sxx, Syy |
| 位相情報 | **使う**（複素数）  | **使わない**（実数のパワーのみ）  |
| 測るもの | 連動の強さ        | 強度の偏り               |

## 5. 計算手順 — 論文 5 ステップの精読

論文に書かれた手順を、現代的な言葉で再表記し、各ステップの意味を解説します。

### Step 1. 時間信号 x(t), y(t) を収集する

2 チャンネル分の時系列データを集める。EEG であれば左右半球の電極から。

サイマトロン系装置のサンプリング周波数 Fs は装置設定で変更可能ですが、**開発元 Elektrika Inc. および標準的な運用では 200 Hz が使用されます**。本資料の以降の記述（FFT 長、bin 幅、帯域 bin 範囲など）はすべて **Fs = 200 Hz 前提** です。

> **注:** Fs を変更して使用する場合、以下のパラメータが連動して変わります：
> - bin 幅 = (Fs / 2) / 128 （デシメーション後の FFT 長 128 で割る）
> - 各帯域に対応する bin 番号
> - 有効ナイキスト周波数 = Fs / 4（デシメーション 1/2 後の Nyquist）
>
> 装置出力と自前計算を照合する場合は、Fs の設定を必ず確認してください。

### Step 2. FFT で周波数領域へ変換

> 論文表記: *"FFT them into X(t) and Y(t)"*  
> ※ これは原文のタイポで、正しくは X(f), Y(f) です。

時間領域の信号を周波数領域に変換するのが FFT (Fast Fourier Transform) です。解析対象の時間信号を一定長の「セグメント」に分割し、各セグメントに窓関数を乗算してから FFT を適用します。

論文には具体的なセグメント長・窓関数の指定はありませんが、サイマトロン系装置では以下の仕様で実装されています。

**サイマトロン系装置の実装仕様**（開発元 Elektrika Inc. 確認済み）：

| 項目 | 値 |
|---|---|
| サンプリング周波数 Fs | 200 Hz（標準運用） |
| 1 セグメント | 256 サンプル（1.28 秒） |
| デシメーション | 256 → 128 サンプル（1/2 デシメ） |
| FFT 長 | 128 点 |
| bin 幅 | 0.78 Hz（= 100 / 128） |
| 使用 bin | 1〜32（DC bin 0 を除外、約 0.78〜25 Hz） |
| 窓関数 | `1 − cos(2πn/127)`, n = 0..127 |
| オーバーラップ | なし |
| ゼロパディング | なし |
| 正規化 | なし |
| 処理順 | DC 除去 → デシメーション → コサイン窓 → FFT |

窓関数 `1 − cos(2πn/127)` は標準的な Hann 窓 `0.5·(1 − cos(2πn/(N−1)))` を振幅2倍したものです（N = 128, 分母 127 は Hann 窓の慣例的な定義に整合）。窓関数はスペクトル漏れ抑制のため適用しますが、開発元によれば正規化は適用しないとのことです。

**デシメーションについての補足**

256 サンプル（200 Hz サンプリングで 1.28 秒）を取得した後、FFT 前に 128 サンプルへデシメーション（1/2 ダウンサンプリング）が行われます。これにより：

- 実効的なナイキスト周波数 = 50 Hz（200 Hz / 2 / 2）
- bin 幅 = 0.78 Hz（≈ 100 Hz / 128）
- bin 1〜32 で約 0.78〜25 Hz をカバー（Total 帯域 0.7〜25 Hz と整合）

> **注:** デシメーション方式（単純な間引きかアンチエイリアスフィルタを伴うか）については開発元から明示されておらず、実装ソースの確認が必要です。単純間引きの場合、25 Hz を超える成分のエイリアシングが発生する可能性があります。

### Step 3. 重要：クロススペクトルは「実部・虚部のまま」平均する

> 原文: *"For coherence, you must use the real and imaginary parts of the cross power spectrum averaged over several segments, not the |Sxy(f)| averaged over those segments. Only after getting the average parts do you do the ℜ² + ℑ² calculation."*

これが論文全体で最も重要な指示です。

**正しい順序**:

$$\overline{S_{xy}}(f) = \frac{1}{M}\sum_{m=1}^{M} S_{xy}^{(m)}(f)$$

$$|\overline{S_{xy}}(f)|^2 = (\operatorname{Re}\overline{S_{xy}})^2 + (\operatorname{Im}\overline{S_{xy}})^2$$

(まず複素数を平均し、最後に絶対値の二乗を取る)

**誤った順序** (絶対NG):

$$\frac{1}{M}\sum_{m=1}^{M} |S_{xy}^{(m)}(f)|^2$$

(各セグメントで絶対値を取ってから平均)

### Step 4. オートスペクトルも同じセグメント数で平均する

$$\overline{S_{xx}}(f) = \frac{1}{M}\sum_{m=1}^{M} S_{xx}^{(m)}(f)$$

$$\overline{S_{yy}}(f) = \frac{1}{M}\sum_{m=1}^{M} S_{yy}^{(m)}(f)$$

### Step 5. 帯域集計は最後に行う

> 原文: *"Note that if you want the coherence for bands, rather than a single bin, you must do the summation at the very end, because the sum of squares (the correct way) is not the same as the square of sums (which is what you'll get if you sum up bins beforehand)."*

帯域（例: Alpha 8–13 Hz）の Coherence を出す場合、論文は「合計は最後に行え」と指示しています。"sum of squares ≠ square of sums" の対比は、セグメント間平均で得られた平均クロススペクトル `S̄xy(k) = S̄xy_re(k) + j·S̄xy_im(k)` の、実部・虚部の処理順序を指しています。

**正しい順序 (sum of squares)** — bin ごとに二乗和を取ってから帯域集計：

$$\sum_{k \in \text{band}} \bigl[\, \overline{S_{xy,\text{re}}}(k)^2 + \overline{S_{xy,\text{im}}}(k)^2 \,\bigr]$$

**誤った順序 (square of sums)** — 実部・虚部を bin にわたって先に足してから二乗：

$$\Bigl[\,\sum_{k \in \text{band}} \overline{S_{xy,\text{re}}}(k)\,\Bigr]^2 + \Bigl[\,\sum_{k \in \text{band}} \overline{S_{xy,\text{im}}}(k)\,\Bigr]^2$$

両者は数学的に異なる値になります。位相がランダムな bin 同士は加算で打ち消し合うため、誤った順序では分子が不当に小さくなり、帯域 Coherence が低く出ます。

なお、帯域 Coherence をどう「集計」するかには複数の流儀があります（bin ごとの γ²(k) を計算してから帯域内で平均・最大を取る、など）。論文は具体的な集計方法は規定していませんが、いずれの流儀でも **「実部・虚部の bin 加算を二乗より先に行わない」** という原則は共通します。具体的な集計方法は 7.2 節を参照。

## 6. 致命的な罠：なぜ「平均化の順序」が全てを決めるか

### 6.1 1 セグメントの Coherence は常に 1

[セクション 3.3](#33-数学的に重要な性質) の通り、1 セグメントだけで Coherence を計算すると **必ず 1.0** になります。

```
1 セグメントの S_xy = X*·Y
|S_xy|^2 = |X|^2 · |Y|^2 = S_xx · S_yy
γ² = S_xx · S_yy / (S_xx · S_yy) = 1
```

### 6.2 「平均後の Coherence」と「Coherence の平均」は別物

| 計算順序 | 結果 |
|---|---|
| **誤**: 各セグメントで γ² を計算 → 平均 | 常に 1.000（数学的恒等式、6.1節参照） |
| **正**: 実虚部のままクロスを累積 → 最後に γ² 計算 | 信号の同期度に応じた値（0〜1） |

順序を間違えると **常に 1.0** という意味のない結果になります。

> **補足:** 「正」の側の具体的な数値は信号の同期度・ノイズ量・セグメント数に依存します。完全同期信号では 1 に近く、独立な雑音同士では 0 に近づきます。詳しい挙動は 6.3 節を参照。

### 6.3 なぜ平均化で意味のある値になるか

複素数の平均はベクトル平均と同じです。

- **2 信号が真に同期していれば**：各セグメントの Sxy は同じ向きを持ち、平均しても向きが残り、絶対値は大きくなる。
- **2 信号が独立なら**：Sxy の向きはセグメントごとにランダム。平均するとベクトルが打ち消し合い、絶対値はほぼ 0 になる。

オートスペクトル Sxx, Syy は常に正の実数なので、向きの問題がなく、独立信号でも同期信号でも値は残ります。

結果：

- 同期信号 → |平均 Sxy|² ≈ 平均 Sxx · 平均 Syy → γ² ≈ 1
- 独立信号 → |平均 Sxy|² ≈ 0 → γ² ≈ 0

これが **位相情報を活かしてセグメント平均する Welch 法** の本質と考えます。

### 6.4 Welch 法のセグメント設計

**一般的な Welch 法のパラメータ**

| パラメータ      | 典型値               | 影響                     |
| ---------- | ----------------- | ---------------------- |
| FFT 長 n    | 256, 512, 1024    | 周波数分解能 = Fs / n        |
| ホップ長 (hop) | n/2 (50% overlap) | セグメント独立性とノイズ低減のバランス    |
| セグメント数 M   | 10〜100            | 多いほど統計的に安定（ただしデータ長が必要） |

一般論としては、50% オーバーラップを採用しセグメント数を増やすことで、統計的安定性とノイズ低減のバランスを取るのが伝統的な手法です。

**サイマトロン系装置の実装**

ただし、サイマトロン系装置の実装は一般論とは異なる設計を採用しています（開発元 Elektrika Inc. 確認済み）：

| 項目 | 装置仕様 | 一般論との差異 |
|---|---|---|
| FFT 長 | 128 点（256 取得 → 128 デシメ） | より短い |
| オーバーラップ | なし | 50% オーバーラップを使わない |
| MSC 用セグメント数 | 直近 8 セグメント（固定） | 固定値・少なめ |

オーバーラップを使わない設計は、MSC のような「持続性」を評価する指標において、各セグメントが独立した時間区間を表すという解釈を優先したものと考えられます（時間的に重なるセグメントを統計に含めると「持続している」という意味が曖昧になるため）。

## 7. 帯域別解析の正しいやり方

EEG では 1 Hz 単位の議論より、生理的に意味のある **周波数帯域** で集計するのが普通です。

### 7.1 標準的な帯域定義

| 帯域    | 範囲         | 生理的意味      |
| ----- | ---------- | ---------- |
| Delta | 0.5 – 4 Hz | 深睡眠、無意識    |
| Theta | 4 – 8 Hz   | 浅睡眠、瞑想、創造性 |
| Alpha | 8 – 13 Hz  | 安静覚醒、リラックス |
| Beta  | 13 – 30 Hz | 活動覚醒、集中、思考 |
| Gamma | 30 Hz 〜    | 高次認知、注意    |

サイマトロン Genie の帯域定義：

| Band   | Range (Hz)  |
| ------ | ----------- |
| Total  | 0.7 – 25.0  |
| Delta  | 0.7 – 3.5   |
| Theta  | 3.5 – 8.0   |
| Alpha  | 8.0 – 13.0  |
| Beta   | 13.0 – 25.0 |
| X Band | 4.0 – 7.5   |
| Y Band | 8.5 – 12.0  |

X Band, Y Band はサイマトロン独自のサブバンドで、Theta 中央域と Alpha 中央域を狙ったもの。

### 7.2 帯域内 Coherence の集計方法

論文の Step 5 に従い、bin 単位で計算してから帯域内で集計します。

```
擬似コード:
for each bin k in band [Lo, Hi]:
    coh[k] = (avgRe[k]² + avgIm[k]²) / (avgSxx[k] · avgSyy[k])

band_max  = max(coh[k] for k in band)
band_mean = mean(coh[k] for k in band)
```

純正アプリの `Coh%` 列は通常 **帯域内最大値** または **帯域内平均値** のいずれか。今回の実装では両方出力しています。

### 7.3 Mean Frequency

帯域内の「重心周波数」も実装可能です：

$$f_{\text{mean}} = \frac{\sum_k f_k \cdot \overline{S_{xx}}(f_k)}{\sum_k \overline{S_{xx}}(f_k)}$$

純正アプリの `Mean Frequencies Hz:` の値はこれです。

### 7.4 Maximum Sustained Coherence (MSC)

**定義（サイマトロン装置マニュアル）**

> "The maximum average coherence value in a 5-second segment during the seizure period."  
> （発作期間中の 5 秒セグメントにおける最大平均コヒーレンス値）

注意：ここでの MSC は **Maximum Sustained Coherence** であり、信号処理一般で言う MSC（Magnitude Squared Coherence = γ²(f) の別名）とは **意味が異なります**。

**装置の実装仕様**（開発元 Elektrika Inc. 確認済み）

1. 治療後（post-treatment）データの **直近 8 セグメント** を常時保持
2. MSC は以下の条件を満たすときのみ計算される：
   - 発作（seizure）が検出されている
   - post-treatment データが 8 セグメント以上集まっている
3. 計算方法：
   - 8 セグメント分の Sxx, Syy のパワー平均で平均オートパワーを算出
   - 8 セグメント分のクロスパワー Sxy の実部・虚部を別々に平均し、最後に |Sxy|² を算出
   - 上記から γ²(f) を計算（論文 Step 3 の原則に準拠）

**「5 秒」の根拠について（未確定）**

装置マニュアルの「5 秒セグメント」と、内部実装の「8 セグメント（1.28 秒 × 8 = 10.24 秒）」の関係は、文献・開発元からの回答からは **現時点で特定できていません**。整合性の検討：

- 200 Hz サンプリングで 5 秒 = 1000 サンプル → 1.28 秒（256 サンプル）の整数倍にならない
- 1.28 秒 × 4 セグメント = 5.12 秒なら自然に整合するが、マニュアル表記は「5 秒」
- 「直近 8 セグメント」と「5 秒セグメント」がどう繋がるかは追加確認が必要

実装時はこの不確実性を念頭に置き、可能であれば装置の出力と自前計算結果を照合することを推奨します。

## 8. 実装時に踏みやすい落とし穴 6 連発

実際にこの論文を実装する過程で遭遇した、典型的なバグの実例集。

### 罠 ① セグメント毎に Coherence を計算してしまう

**症状**: 結果が常に 1.000 (100%) になる。

**原因**: [セクション 6.1](#61-1-セグメントの-coherence-は常に-1) のとおり数学的恒等式。

**修正**: 累積器パターンを使う。

```csharp
// NG: 誤
foreach (segment) {
    coh = ComputeCoherence(segment);
    cohSum += coh;
}
result = cohSum / segCount;

// OK: 正
foreach (segment) {
    Accumulate(Sxx, Syy, Sxy_re, Sxy_im);  // 生スペクトルを足す
}
result = ComputeCoherenceFromAccumulated();  // 最後に1回だけ計算
```

### 罠 ② FFT twiddle テーブルの符号バグ

**症状**: 純粋正弦波を FFT に入れても、本来立つべき bin にエネルギーが集まらず散らばる。

**原因**: `cos` テーブルを半周期分しか持たず、剰余演算で第二半周期の符号反転を取り損なう。

```csharp
// NG: 半周期テーブル + % (MAXSAMP/2)
private int[] cosTbl = new int[MAXSAMP / 2];
int wi = cosTbl[(MAXSAMP/4 + k*delta) % (MAXSAMP/2)];  // 符号情報損失

// OK: 全周期テーブル + % MAXSAMP
private int[] cosTbl = new int[MAXSAMP];
int wi = cosTbl[(MAXSAMP/4 + k*delta) % MAXSAMP];
```

または、`cos` と `sin` を別テーブルにする（推奨）。

### 罠 ③ Nyquist bin の取りこぼし

**症状**: 最高周波数 (Fs/2) のパワーが常に 0。

**原因**: ループが `i < n/2` で終わり、`i = n/2` の bin を計算していない。

```csharp
// NG:
for (int i = 0; i < n/2; i++) { ... }

// OK
for (int i = 0; i <= n/2; i++) { ... }  // <= に変更
```

### 罠 ④ 整数 FFT のオーバーフロー

**症状**: FFT 後の値が異常に小さく、Coherence 計算で結果が ~0 になる。

**原因**: `int * int` で 32bit 上限を超え、ラップアラウンドで値が破壊される。

```csharp
// NG: int * int でオーバーフロー
int xr = (real[l] * wr - imag[l] * wi) / ZOOM;

// OK: long 経由で計算
long mr = (long)real[l] * wr - (long)imag[l] * wi;
int xr = (int)(mr / ZOOM);
```

または、最初から `double` にしてしまうのが最も安全。

### 罠 ⑤ サンプリング周波数・デシメーションの取り違え

**症状**: 帯域定義 (8–13 Hz など) の bin インデックスがずれて、別の周波数を集計してしまう。

**原因**: Fs の値、または FFT 前のデシメーションの有無を取り違えている。

**サイマトロン系装置の正しい関係式**：

```
取得サンプリング周波数 Fs       = 200 Hz
取得サンプル数 / セグメント      = 256
                ↓ 1/2 デシメーション
FFT 入力サンプル数              = 128
有効サンプリング周波数          = 100 Hz
bin 幅                          = 100 / 128 ≈ 0.78 Hz
有効ナイキスト周波数            = 50 Hz
使用 bin                        = 1〜32（DC を除外、約 0.78〜25 Hz）
```

**チェック方法**: 純正アプリのスペクトル表示で bin 幅を確認し、上記の関係式と整合するか確かめる。

なお、デシメーションを忘れて 256 点 FFT を実行した場合、bin 幅は同じく約 0.78 Hz（= 200/256）になるため帯域 bin の範囲は一見一致しますが、以下の点で装置と差が出ます：

- 実効ナイキスト周波数（デシメなし 100 Hz vs デシメあり 50 Hz）
- アンチエイリアスフィルタの有無による 25 Hz 超成分の扱い
- 窓関数を適用するサンプル点数（128 vs 256）

装置出力との数値照合を行う場合は、データ点数・処理パイプラインを装置仕様に完全一致させる必要があります。

### 罠 ⑥ 過剰なスムージング（論文・装置仕様外の一般的注意）

> **注**: この項目は論文・開発元から指示されたものではなく、実装上の一般的な経験則です。罠①〜⑤と異なり、論文の指示違反ではありません。

**症状**: 計算ロジックは正しいのに、ピーク値が予想より低い。

**原因**: 3点移動平均などのポストプロセスが、シャープな共鳴ピークを潰している可能性がある。

```csharp
// 例: 隣接 bin の平均でスムージング
cohSmooth[k] = (coh[k-1] + coh[k] + coh[k+1]) / 3.0;
```

サイマトロン系装置の FFT 長は 128 固定であり、bin 幅 0.78 Hz は EEG 解析として十分細かいため、追加のスムージングは通常不要です。装置の出力と自前計算結果を照合する場合、装置側がスムージングを行っているか不明なので、まずは **スムージングなし** で実装し、結果が一致するか確認することを推奨します。

## 9. Genie 純正アプリの Bands 表との対応

サイマトロン Genie の `Bands EEG 1&2 (accumulated)` 画面は、本論文の指標を以下の列で表示しています。

```
Frequency Hz   Absolute μV²    Relative %    Asym %    Coh %
              EEG1   EEG2     EEG1   EEG2
Total  0.7-25.0  ...    ...    ...    ...    ...      ...
Delta  0.7- 3.5  ...    ...    ...    ...    ...      ...
Theta  3.5- 8.0  ...    ...    ...    ...    ...      ...
Alpha  8.0-13.0  ...    ...    ...    ...    ...      ...
Beta  13.0-25.0  ...    ...    ...    ...    ...      ...
X Band 4.0- 7.5  ...    ...    ...    ...    ...      ...
Y Band 8.5-12.0  ...    ...    ...    ...    ...      ...
Mean Frequencies Hz:    ...    ...
Segments: Collected-NN   Computed-NN
```

各列の計算式：

| 列                       | 計算式                                    |
| ----------------------- | -------------------------------------- |
| **Absolute μV²**        | 帯域内総和 平均 Sxx(f\_k) (and Syy)           |
| **Relative %**          | (帯域パワー / Total 帯域パワー) × 100            |
| **Asym %**              | A(f) の帯域内平均または最大 × 100                 |
| **Coh %**               | γ²(f) の帯域内平均または最大 × 100                |
| **Mean Frequencies Hz** | Σ f\_k · 平均 Sxx(f\_k) / Σ 平均 Sxx(f\_k) |
| **Collected**           | FFT 投入セグメント数                           |
| **Computed**            | アーチファクト除外後に有効に使われたセグメント数               |

> **Collected と Computed の差について（補足）**  
> Bands 画面の `Segments: Collected-NN Computed-NN` で値が異なる場合、両者の差分は何らかの理由でセグメントが解析対象から除外された数を表します（アーチファクト除外、装着/取り外し時の無駄区間カット等）。具体的な除外ロジックは装置・実装ごとに異なるため、本論文の範囲外です。

## 10. 結果の解釈ガイド

### 10.1 健常な覚醒安静の典型値

| 項目                | 期待値             |
| ----------------- | --------------- |
| Alpha Coh %       | 70–95%（高同期）     |
| Alpha Asym %      | 0–30%（低非対称）     |
| Alpha Relative %  | 30–60%（覚醒時はα優位） |
| Mean Frequency Hz | 9–11（α帯重心）      |

### 10.2 注意すべきパターン

**Delta が異常に大きい (Relative > 50%)**

- 深睡眠状態 OR
- DC ドリフトやアーチファクトの混入 OR
- ハイパスフィルタ未適用

**Coh % が全帯域で低い (mean < 30%)**

- 電極接触不良
- 片側電極の信号品質低下
- 真に左右が独立した活動状態

**Asym % が特定帯域だけ突出**

- その帯域の生成源が片側半球に偏在
- 片側電極のアーチファクト

### 10.3 Coherence × Asymmetry のマトリクス解釈

|                    | 低 Asym (< 30%) | 中 Asym (30–70%) | 高 Asym (> 70%) |
| ------------------ | -------------- | --------------- | -------------- |
| **高 Coh (> 70%)**  | 理想的な左右協調       | 同期下の偏在活動        | 一側性活動の強い同期     |
| **中 Coh (30–70%)** | 部分的協調、健全範囲     | 典型的な日常状態        | 一側優位の部分同期      |
| **低 Coh (< 30%)**  | 左右独立、対称        | 左右独立、軽度偏在       | 強い一側性活動        |

## 11. 付録：用語集

| 用語                 | 説明                                                                  |
| ------------------ | ------------------------------------------------------------------- |
| **FFT**            | Fast Fourier Transform。時間信号を周波数成分に分解するアルゴリズム。                       |
| **bin**            | FFT 出力の 1 つの周波数点。bin 幅 = Fs/n。                                      |
| **Nyquist 周波数**    | Fs/2。これより高い周波数は理論的に再現不可能。                                           |
| **Welch 法**        | 信号を複数のセグメントに分け、それぞれをFFTして平均することで安定なスペクトル推定を得る手法。                    |
| **MSC**            | **Maximum Sustained Coherence** の略。サイマトロン系装置で使用される指標で、発作期間中の最大持続コヒーレンス値。信号処理一般で MSC と呼ばれる Magnitude Squared Coherence（γ²(f) の別名）とは意味が異なるので注意。詳細は 7.4 節参照。 |
| **窓関数**            | FFT 前に信号に掛ける重み関数。両端を 0 に近づけて leakage を抑制。代表例：Hann、Hamming、Blackman。 |
| **leakage**        | FFT が信号を周期と仮定することによる、本来の周波数からのエネルギー漏れ。                              |
| **Auto-spectrum**  | 1 信号のパワースペクトル (Sxx, Syy)。常に正の実数。                                    |
| **Cross-spectrum** | 2 信号間のスペクトル (Sxy)。複素数で位相情報を含む。                                      |
| **オーバーラップ**        | Welch 法でセグメントを部分的に重ねること。50% が標準。                                    |
| **アーチファクト**        | 脳活動以外の混入信号。瞬き、体動、筋電、商用電源など。                                         |
| **ハイパスフィルタ**       | 低周波（DC ドリフトなど）をカットするフィルタ。EEG では 0.5 Hz 程度を境にすることが多い。                |
